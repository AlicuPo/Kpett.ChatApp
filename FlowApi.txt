Kpett.ChatApp API Flow
Cap nhat tu source code hien tai trong workspace.

========================================================================
1. Quy uoc chung
========================================================================

Base path
- API dung ASP.NET Controller routing.
- Header cho endpoint can dang nhap: Authorization: Bearer <access_token>

JSON casing
- DTO C# dung PascalCase, nhung JSON tra ra tu ASP.NET se o dang camelCase.

Success wrapper thong dung
- GeneralResponse
  {
    "isSuccess": true,
    "message": "...",
    "statusCode": 200
  }

- GeneralResponse<T>
  {
    "isSuccess": true,
    "message": "...",
    "statusCode": 200,
    "data": { ... }
  }

- PaginatedData<T>
  {
    "items": [ ... ],
    "pagination": {
      "nextCursor": "base64-or-null",
      "hasMore": true,
      "totalCount": 123,
      "limit": 10
    }
  }

Error wrapper logic
- Phan lon business error di qua GlobalExceptionHandler va co y dinh tra:
  {
    "isSuccess": false,
    "statusCode": 400,
    "errorCode": "VAL.REQUIRED",
    "message": "...",
    "stackTrace": "..." // chi co o Development
  }
- JWT middleware co 2 nhanh rieng:
  - 401 AUTH.ACCESS_TOKEN_INVALID khi token sai/het han
  - 403 AUTH.FORBIDDEN khi user da xac thuc nhung khong du quyen
- OptionalAuthorize endpoint:
  - Neu khong gui token: van vao duoc.
  - Neu gui Bearer token sai/het han: 401 AUTH.ACCESS_TOKEN_INVALID.

Luu y implementation hien tai
- Nhieu endpoint tra HTTP 200 nhung body.statusCode = 201.
- /api/media/delete dang tra body.statusCode = 2000.
- Mot so endpoint khong dung GeneralResponse ma tra raw object/list.
- GlobalExceptionHandler hien dang WriteAsJsonAsync(jsonString), nen thuc te body loi business co the bi double-serialize thanh chuoi JSON escaped. Ve mat flow, cac loi duoi day van duoc mo ta theo ErrorResponse mong muon.
- Co endpoint khong gan [Authorize] nhung van goi User.GetRequiredUserId(), nen thuc te van can user claim de chay duoc.

DTO tom tat hay gap
- UserResponse: id, username, email, displayName, avatarUrl, isProfileCompleted, isVerified, lastActiveAt?, createdAt?, updatedAt?
- UserGeneralInfoResponse: UserResponse + coverUrl, dateOfBirth?, biography?, occupation?, location?
- UserProfileResponse: UserResponse + dateOfBirth?, biography?, occupation?, location?, coverUrl?, stats?, viewerContext?, isOnline
- TokenResponse: accessToken, refreshToken, tokenType
- ConversationResponse: id, name, avatarUrl, type, lastMessageAt, lastMessage?, unreadCount
- MessageDTO: id, content, createdAt, senderId, type, metadata
- MessagePageResult: messages[], oldestMessageId?, hasMore
- PostFeedResponse: id, author, title?, content?, hashtags[], media[], metrics, viewerContext, privacy, createdAt, updatedAt?
- PostThumbnailResponse: PostFeedResponse + mediaThumbnail?
- PostReactionDTO: id, postId, userId, type, createdAt
- CommentListItemDTO: id, postId, parentId?, author, content, mentions[], attachments[], metrics, viewerContext, isEdited, isDeleted, createdAt, updatedAt?
- FriendRequestResponse: requestId, senderId, receiverId, status, createdAt
- FriendListItemDTO: id, username, displayName, avatarUrl, isVerified, isOnline, friendedAt?
- NotificationResponse: id, type, referenceId?, metadata?, isRead, createdAt, actor?, sound
- NotificationSoundResponse: enabled, key, volume
- CloudinarySignatureResponse: publicId, signature, timestamp, cloudName, apiKey, folder, uploadUrl, notificationUrl, tags
- MediaUploadResponse: publicId, secureUrl, format, bytes

2. Auth APIs

POST /api/auth/login
Auth
- Public

Request
- Body
  {
    "email": "user@example.com",
    "password": "secret",
    "deviceToken": "optional",
    "deviceType": "optional"
  }

Response
- HTTP 200
- Body: GeneralResponse<LoginResponse>
  - data.user: UserResponse
  - data.token: TokenResponse

Errors
- 400 VAL.REQUIRED: Email not empty
- 401 AUTH.UNAUTHORIZED: Wrong email or password
- 403 USER.INACTIVE: User inactive
- 500 SERVER.SYSTEM_ERROR

POST /api/conversations
Auth
- Required

Request
- Body

    "type": "Direct|Group",
    "participantIds": ["current-user-id", "other-user-id"],
    "initialMessage": "optional initial text",
    "name": "optional group name"
  }
- Direct conversation can co dung 2 participant va initialMessage.
- Group conversation can name va toi thieu 2 participant.

Response
- HTTP 201
- Body: GeneralResponse<ConversationResponse>

Errors
- 400 VAL.REQUIRED:
  - Conversation request and type are required.
  - Initial message content is required to create a direct conversation.
  - Group name is required.
- 400 VAL.INVALID:
  - Invalid conversation type. Must be 'Direct' or 'Group'.
  - Direct conversation must have exactly 2 different participants.
  - A group must have at least 2 participants.
- 401 AUTH.ACCESS_TOKEN_INVALID
- 401 AUTH.UNAUTHORIZED: User is not authenticated.
- 404 USER.NOT_FOUND: One or more participants do not exist.
- 500 SERVER.SYSTEM_ERROR
